## Project Overview

Interactive Bash script tool that deploys a proxy server stack (Xray + Nginx + Sing-Box) on Linux. All user-facing UI is in Chinese. Run as root via `bash install.sh` or the curl one-liner. There is no build system, test suite, or CI — validation is manual on a clean VPS.

## Running

```bash
sudo bash install.sh          # interactive menu system
# Option 0 = full auto-install through all steps
# Option s = sync modules to local cache
# Option u = uninstall menu
```

## Architecture: Module Loading and State

**Entry point**: `install.sh` — the only runnable script. It sources modules on demand via `load_module()`.

**Module resolution order** (in `load_module`):
1. Local cache: `/etc/xray-deploy/modules/<name>.sh`
2. Bundled: `modules/<name>.sh` (repo directory)
3. Remote download from `BASE_URL` → cached locally

**State file**: `/etc/xray-deploy/config.env` — shell-sourceable `KEY='value'` file. All cross-module data flows through `get_state()` / `save_state()` in install.sh. Modules communicate via shell globals hydrated from this file, not by calling each other directly.

## Module Dependency Chain

```
system (1) → unbound (2) → nginx install (3) → cert (4)
                                ↳ xray install (5) → WARP (w) → xray config (8) → sync nginx routes
                                ↳ singbox install (6) ────────→ singbox config (9) → sync nginx routes
```

`modules/sync.sh` is the cross-module glue: it restores domain arrays from state, hydrates client state by parsing actual Xray/Sing-Box JSON configs with python3, and refreshes nginx routes after reconfiguration.

## Traffic Flow (Port Architecture)

```
:443  Nginx stream (SNI routing)
  ├── Reality domains    → :9443  (Xray Reality, acceptProxyProtocol)
  ├── XHTTP domain       → :20443 (nginx http → Xray :8001)
  ├── gRPC domain        → :20445 (nginx http → Xray :8002)
  ├── AnyTLS domain      → :18443 (nginx proxy_protocol → Sing-Box :8443)
  ├── Reality fallback   → :10080 (Xray → nginx fallback server, proxy_protocol)
  └── Unknown SNI        → :20880 (trap server, self-signed cert)
```

## Critical State Variables

Domain arrays are stored as space-separated strings and restored with `read -ra`:
- `ALL_DOMAINS`, `CDN_DOMAINS`, `DIRECT_DOMAINS`, `REALITY_SERVER_NAMES`

Installation flags (`INST_*`, `CONF_*`): set to "1" on completion. `_sync_inst_state()` auto-detects installed services and backfills missing flags.

Crypto material (`XRAY_UUID`, `XRAY_PUBLIC_KEY`, `XRAY_PRIVATE_KEY`, `SINGBOX_PASSWORD`, `WGCF_*`): persisted across re-runs; only regenerated if absent from state.

## Known State Bugs (Fixed)

- `XHTTP_PATH` must be saved immediately on generation, before nginx config runs, or nginx and xray get different paths
- `REALITY_SERVER_NAMES` must be saved/restored as the complete array, not just the first element, or the nginx SNI map will be incomplete
- WGCF credentials must be saved by `install.sh` (not `warp.sh`) to avoid format conflicts

## Key Conventions

- `set -euo pipefail` at the top of every script
- Functions: `snake_case` for public, `_prefixed_snake_case` for private helpers
- JSON manipulation uses `python3` subprocesses (not bash-native)
- All generated config files include "Auto-generated by xray-nginx-deploy" headers with timestamps
- Hardware profiles (small/medium/large) drive nginx and sysctl tuning based on CPU/memory detection
- WARP integration uses embedded WireGuard outbounds in both Xray and Sing-Box configs (not a local SOCKS5 proxy)
- `BASE_URL` in install.sh points to the active development branch (`codex/proxy-kernel-service-tuning`), not `main`

## Supported Protocols

| Protocol | Transport | Link Format |
|----------|-----------|-------------|
| VLESS+Reality | Direct TCP+TLS | `vless://UUID@IP:443?security=reality&...` |
| VLESS+XHTTP | CDN (Cloudflare) | `vless://UUID@domain:443?type=xhttp&path=...` |
| VLESS+gRPC | CDN (Cloudflare) | `vless://UUID@domain:443?type=grpc&serviceName=...` |
| AnyTLS | Direct | `anytls://password@domain:443?security=tls` |

## Generated Config Files

State: `/etc/xray-deploy/config.env`
System: `/etc/sysctl.d/99-xray-optimize.conf`, `/etc/security/limits.d/99-xray-optimize.conf`
DNS: `/etc/unbound/unbound.conf`, `/etc/unbound/conf.d/`
Nginx: `/etc/nginx/nginx.conf`, `/etc/nginx/conf.d/00-upstreams.conf`, `fallback.conf`, `servers.conf`, `/etc/nginx/cloudflare_real_ip.conf`
Certs: `/etc/letsencrypt/live/<root>/`, `/etc/cloudflare/`
Xray: `/usr/local/etc/xray/config.json`
Sing-Box: `/etc/sing-box/config.json`
WARP: `/etc/wgcf/wgcf-profile.conf`
Cron: `/usr/local/bin/update_cf_ip.sh` (weekly), `/usr/local/bin/update-root-hints.sh` (monthly systemd timer)
